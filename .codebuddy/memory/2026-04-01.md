## 修复早报生成失败：AI 返回 JSON 解析崩溃
- **问题**：手动触发早报后报错 `'str' object has no attribute 'get'`，出在 `ai_analyzer.py` 第 1426 行解析 `highlights` 时
- **根因**：早报 prompt 中 `highlights` 只写了 `[...]` 未展开字段说明，DeepSeek 返回了字符串数组（如 `["标题1", ...]`）而非对象数组（如 `[{"title": "xxx", ...}]`）
- **修复**（`backend/app/services/ai_analyzer.py`）：
  1. 早报 prompt 补充 `highlights` 和 `watchlist_analysis` 的完整 JSON 格式示例
  2. `highlights` 解析增加 `isinstance` 类型判断：字符串自动转 NewsHighlight
  3. `cross_border_events` 解析增加类型判断：字符串自动跳过
  4. `market_analysis` 解析增加容错：非字典类型使用默认值
- **已推送**：GitHub + Gitee，需到服务器 `git pull` 并重启 uvicorn
