# 2026-03-31 工作日志

## 新增市场数据增强模块

在 `market_data_service.py` 中新增了三大数据采集模块：

### 1. 概念板块排行 (`ConceptSectorData`)
- 使用 `ak.stock_board_concept_name_em` 获取东方财富概念板块行情
- 按涨跌幅排序取Top10
- 新增方法：`_get_concept_sectors()`

### 2. 连板强势股 (`ConsecutiveLimitStock`)
- 使用 `ak.stock_zt_pool_em` 涨停股票池接口，筛选连板>=2天的股票
- 按连板天数降序排列
- 新增方法：`_get_consecutive_limit_stocks()`

### 3. 技术信号机会股 (`TechSignalStock`)
- **路径A**：同花顺技术选股接口（量价齐升、连续上涨、创新高、持续放量）
  - 方法：`_get_ths_tech_stocks()`
- **路径B**：自研多因子技术信号扫描（初筛全A股→Top20拉K线→8大信号打分）
  - 方法：`_scan_tech_signals()`
  - 信号包括：MACD金叉、均线多头排列、突破20日均线、KDJ金叉、RSI超卖反弹、布林带突破中轨、放量、连续上涨
  - 至少触发2个信号才入选
- 合并方法：`_get_tech_signal_stocks()`（两路径合并去重，双路径共振加分）

## 修复报告详情页白屏问题

### 第一轮修复
**根因**：`fc0d050` 提交中删除了 `handleShare` 函数定义，但遗漏了第184行对该函数的引用（分享按钮的 `onPress={handleShare}`）。React Native 运行时遇到未定义函数引用直接崩溃，导致白屏。

**修复**：删除残留的分享按钮 `<IconButton>` 组件（5行代码）。

### 第二轮修复（全面加固）
白屏仍存在，进行全面排查，发现多个 null 安全隐患：

1. **`report.analysis.opportunities.length`** — `opportunities` 可能为 undefined/null，直接 `.length` 崩溃
2. **`report.analysis.risks.length`** — 同上
3. **`<Markdown>{report.content}</Markdown>`** — content 可能为 null，Markdown 组件无法处理
4. **`report.core_opinions.length`** 和 **`report.cross_border_events.length`** — 可能不是数组

**修复措施**：
- 对 `opportunities`/`risks` 加前置 `&&` 检查
- 对 `content` 加 `|| ''` 兜底
- 对 `core_opinions`/`cross_border_events` 改用 `Array.isArray()` 检查
- 新增 `ReportErrorBoundary` 类组件包裹整个页面，捕获所有渲染错误并显示友好提示，彻底防止白屏

**文件**：`mobile/app/(tabs)/report/[id].tsx`

## 股票搜索逻辑重写（实时调接口）

**问题**：搜索"东土"等非热门股搜不到。旧逻辑是缓存优先，缓存为空时只从内置60只股票中搜，后台异步更新缓存但不等结果。

**新逻辑**：
1. 缓存有效（存在且<1小时）→ 直接从缓存搜（极快）
2. 缓存无效 → **同步调用 AKShare 拉取全市场数据**（A股 `stock_zh_a_spot_em` + 港股 `stock_hk_spot_em`），等数据回来后再搜索（首次约3-5秒）
3. 接口也失败 → 降级到旧缓存 → 内置列表

**前端配套**：
- 搜索 API 超时从 15s 增加到 30s
- 加载提示改为"正在搜索全市场股票... 首次搜索需加载数据，请稍等几秒"

## 搜索方案再次重写（东方财富搜索API）

**问题**：AKShare `stock_zh_a_spot_em()` 在服务器上同步调用失败（可能超时或被限流），导致搜索始终降级到内置60只股票。

**最终方案**：
- 完全抛弃 AKShare 全量拉取方案
- 改用**东方财富搜索API**：`https://searchapi.eastmoney.com/api/suggest/get`
- 优势：毫秒级响应、支持中文/代码/拼音搜索、覆盖全A股+港股
- 过滤逻辑：只保留 `Classify == "AStock"` 和 `Classify == "HK"`（排除权证、债券等）
- 港股额外过滤掉牛熊证/权证（名称含"购/沽/牛/熊/法兴/中银"等）
- 降级：API 失败时用内置列表兜底
- 使用 `httpx.AsyncClient` 异步调用（已在 requirements.txt 中）
