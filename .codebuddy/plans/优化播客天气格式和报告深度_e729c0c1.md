## 用户需求

### 需求一：修复天气播报

当前播客中天气播报效果差，会念英文天气描述和"KMH"等单位。用户要求天气播报格式统一为：
"北京今天天气x，气温x摄氏度到x摄氏度，空气质量指数x"

需要新增空气质量指数数据的获取。

### 需求二：优化报告和播客的内容质量

当前报告和播客内容过于泛泛，缺乏投资指导意义（"你不说我也知道"）。需要：

- 报告风格：两者兼顾（投资决策导向 + 深度分析），但精简模块
- 播客时长：控制在10-15分钟
- 核心目标：让内容从"描述已知信息"转变为"提供可操作的投资决策建议"

## 产品概述

对现有每日财经播报系统的天气播报和核心内容生成进行优化升级，提升播客和报告的实用价值。

## 核心特性

1. **天气播报重构**：确保天气描述始终为中文，新增空气质量指数，输出固定格式的简洁天气播报
2. **报告模块精简**：从7个模块精简为5个，每个模块更加深入，重点突出投资决策建议
3. **Prompt重写**：增强"该怎么操作"的决策导向，减少泛泛描述，增加前瞻性预判和具体操作建议
4. **播客脚本优化**：字数目标从3000-6000字调整为2500-3750字（对应10-15分钟），更口语化、更有观点

## 技术栈

- 后端：Python + FastAPI（现有项目架构）
- 异步HTTP：httpx（现有依赖）
- TTS：Edge TTS（现有依赖）
- AI：Claude/DeepSeek/OpenAI（现有多模型链路）

## 实现方案

### 总体策略

本次优化涉及3个核心文件的修改，不引入新依赖，不改变架构，仅优化数据获取逻辑和Prompt内容：

1. 重写天气获取方法，增加空气质量API，确保中文输出
2. 重写报告生成Prompt，从7模块精简为5模块，强化决策导向
3. 调整播客脚本生成的字数控制和口语化表达

### 关键技术决策

**天气API选择**：

- 主API保持 `wttr.in`（`format=j1`），但增加健壮的中文回退逻辑——当 `lang_zh` 字段为空时，使用内置的英文到中文天气映射字典，而非直接使用英文描述
- 空气质量：使用免费的 `aqicn.org` API（`https://api.waqi.info/feed/beijing/?token=demo`）获取北京AQI，该API返回JSON格式，包含aqi数值，无需注册即可使用demo token
- 备用方案：若AQI获取失败，天气播报中省略空气质量部分，不影响整体播报

**报告模块精简方案**（7模块 -> 5模块）：

- 保留并强化：核心判断（模块1）、大盘+资金面分析（模块2，合并原模块2/3）、自选股专属分析（模块3，提到前面作为重点）、事件驱动+操作策略（模块4）、风险雷达+仓位建议（模块5）
- 删除/合并：原模块3（行业轮动）的精华并入模块2；原模块4（热门个股）的核心内容并入自选股分析
- 这样每个模块的AI输出空间更大，分析可以更深入

**Prompt优化核心原则**：

- 在每个模块的要求中加入**"所以呢"约束**——要求AI在每段分析后必须给出"对你的操作意味着什么"
- 核心判断从"描述型"改为"决策型"——示例格式从"创业板涨了2.3%"改为"建议明天加仓xx，理由是..."
- 增加**"反共识观点"要求**——要求AI识别市场一致预期中的盲点，而非重复市场共识
- 自选股分析要求给出**明确的买/卖/持有信号 + 目标价位 + 止损位**

**播客字数调整**：

- 目标字数从3000-6000改为2500-3750（中文TTS约250字/分钟，对应10-15分钟）
- 截断阈值从6000降为3750，截断后保留结束语
- 报告正文精华提取从前2000字降为前1500字

## 实现要点

### 天气方法重写细节

1. `get_beijing_weather()` 方法拆分为两步：获取天气 + 获取AQI
2. 新增英文天气到中文的映射字典（覆盖 wttr.in 常见的30+种天气描述）：

```
"Sunny" -> "晴", "Partly cloudy" -> "多云", "Overcast" -> "阴", "Rain" -> "雨" ...
```

3. 最终输出格式固定为：`"北京今天天气{desc}，气温{min}摄氏度到{max}摄氏度，空气质量指数{aqi}"`
4. AQI获取失败时：`"北京今天天气{desc}，气温{min}摄氏度到{max}摄氏度"`
5. 全部失败时：`"北京今天天气不错"`

### Prompt重写要点

角色设定增加一条：**"你是用户的私人投资顾问，每一段分析都必须回答'所以我该怎么做'"**

5个模块的核心要求：

- 模块1（操作指南）：3条明确的操作建议（不再叫"核心判断"），每条包含"做什么+为什么+风险是什么"
- 模块2（市场全景）：大盘+行业轮动合并，重点是资金流向和板块切换的操作含义
- 模块3（自选股作战图）：每只股票给出信号/目标价/止损位/仓位建议
- 模块4（事件催化）：未来一周的事件+对应的埋伏策略
- 模块5（风控仪表盘）：系统性风险+总仓位建议+需要回避的雷区

写作风格约束新增：

- 禁止使用"值得关注"、"需要注意"等模糊表述，必须说"建议买入/卖出/观望"
- 每个观点后必须有"操作建议"一句话

### 播客脚本优化

- `_prepare_podcast_text` 中字数阈值从6000改为3750
- `_extract_key_paragraphs` 中截取上限从2000改为1500
- 开场白中删去"废话不多说"等套话，更直接地进入核心内容
- 核心观点部分的过渡语更口语化

- `ai_analyzer.py`：AI生成报告环节的Prompt
