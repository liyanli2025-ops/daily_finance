## 🔴 一、这次线上出错的根本原因分析

### 1.1 问题本质：**数据库结构不同步**

| 位置 | 代码版本 | 数据库版本 | 结果 |
|------|---------|-----------|------|
| 本地 | 最新 | 最新（你一直在用） | ✅ 正常 |
| 服务器 | 最新（刚拉取） | **旧的**（之前生成的） | ❌ 报错 |

**通俗解释**：
- 你的代码就像"菜谱"，数据库就像"厨房的架子"
- 新菜谱需要用到"调料架第5格"（`core_opinions` 字段）
- 但服务器的老厨房只有4格架子
- 所以做菜时找不到第5格，就报错了

**为什么本地没问题？**
- 因为你本地一直在开发，数据库是跟着代码一起更新的
- 但服务器的数据库是很早之前创建的，后来代码新增了字段，数据库没跟上

### 1.2 我的错误指导（反思）

| 错误操作 | 正确做法 |
|---------|---------|
| ❌ 让你一个一个字段去加 `ALTER TABLE` | ✅ 应该直接删除旧数据库，让代码自动创建新的 |
| ❌ 没有第一时间查看完整错误日志 | ✅ 出问题先看日志 `journalctl -u finance-daily -n 50` |
| ❌ 路径记错（`backend/main.py` vs `backend/app/main.py`） | ✅ 先确认文件结构再操作 |

### 1.3 根本解决方案

**以后代码更新后，如果涉及数据库结构变化：**

```bash
# 在服务器上执行
cd /var/www/finance-daily/backend

# 1. 备份旧数据库（保留历史数据）
mv data/database.db data/database.db.bak

# 2. 重启服务（自动创建新数据库）
sudo systemctl restart finance-daily
```

## 🟡 三、日常代码更新流程（必读！）

### 3.1 场景一：在当前电脑修改代码后发布

```
步骤1：修改代码
    ↓
步骤2：UGit 提交并推送到 GitHub（必须！）
    ↓
步骤3：(可选) 同步到 Gitee
    ↓
步骤4：登录服务器，拉取最新代码
    ↓
步骤5：重启服务
```

#### 详细操作：

**步骤 2：UGit 推送到 GitHub**
1. 打开 UGit
2. 选择 finance-daily 项目
3. 查看"更改"列表，确认修改内容
4. 填写提交信息（如："修复了XX bug"）
5. 点击"提交"
6. 点击"推送" → 选择 `origin`（GitHub）

**步骤 3：(可选) 同步到 Gitee**
```bash
# 方法1：在 UGit 中推送
# 推送时选择 "gitee" 而不是 "origin"

# 方法2：命令行（在项目目录下）
git push gitee master
```

**步骤 4-5：服务器更新**
```bash
# SSH 登录服务器后执行：
cd /var/www/finance-daily
git pull origin master
sudo systemctl restart finance-daily
```

## 🔵 四、关键问题解答

### Q1: UGit 部署到 GitHub 还要不要做？
**必须做！** 这是代码同步到服务器的唯一通道。

流程：本地 → GitHub → 服务器

### Q2: Gitee 要不要同步？
**建议同步，但不强制。**

- **同步的好处**：国内访问快，多一个备份
- **不同步的问题**：不会影响服务器（因为服务器从 GitHub 拉）
- **建议**：每次推送到 GitHub 后，顺手也推送到 Gitee

### Q3: 两个仓库不同步会导致服务器混乱吗？
**不会！** 因为服务器只认 GitHub。

```
GitHub ← 服务器只从这里拉代码
Gitee  ← 服务器不管这个
```

只要保证 GitHub 是最新的，服务器就能正常更新。

### Q4: 服务器从哪里下载代码？
```bash
# 服务器执行：
cd /var/www/finance-daily
git pull origin master
```
`origin` 就是 GitHub，代码从 GitHub 下载。

### Q5: 另一台电脑的代码放哪个文件夹？
**随便你选，但建议统一风格：**

- Windows: `D:\Projects\daily_finance\` 或 `C:\Users\你的用户名\Desktop\daily_finance\`
- Mac: `~/Projects/daily_finance/`

**关键点**：文件夹名字无所谓，里面的内容要从 GitHub 克隆下来。

## 🟣 五、常见错误及解决方案

### 错误1：数据库字段缺失
```
sqlalchemy.exc.OperationalError: no such column: xxx
```

**解决**：
```bash
cd /var/www/finance-daily/backend
mv data/database.db data/database.db.bak
sudo systemctl restart finance-daily
```

### 错误2：推送冲突
```
error: failed to push some refs
```

**解决**：
```bash
git pull origin master  # 先拉取
# 如果有冲突，解决冲突后
git add .
git commit -m "解决冲突"
git push origin master
```

### 错误3：服务启动失败
```bash
# 查看详细错误
journalctl -u finance-daily -n 100 --no-pager

# 常见原因：
# 1. Python 依赖没装 → pip install -r requirements.txt
# 2. 数据库问题 → 删除重建
# 3. 端口被占用 → 换端口或杀进程
```

### 错误4：文件权限问题
```bash
# 给予执行权限
chmod +x /var/www/finance-daily/backend/run.sh

# 更改文件所有者
chown -R www-data:www-data /var/www/finance-daily
```

### 错误5：ffmpeg/ffprobe 找不到（播客生成失败）
```
FileNotFoundError: [Errno 2] No such file or directory: 'ffprobe'
```

**原因**：systemd 服务的 PATH 环境变量没有包含 `/usr/bin`，导致 pydub 找不到系统安装的 ffmpeg/ffprobe。

**解决**：修改 systemd 服务配置文件，添加系统命令路径到 PATH：
```bash
sudo nano /etc/systemd/system/finance-daily.service
```

确保 `Environment` 行包含 `/usr/bin`：
```ini
Environment="PATH=/usr/bin:/usr/local/bin:/var/www/finance-daily/backend/venv/bin"
```

保存后重载并重启：
```bash
sudo systemctl daemon-reload
sudo systemctl restart finance-daily
```

## 🟠 六、systemd 服务配置（重要！）

服务器使用 systemd 管理后端服务，配置文件路径：
```
/etc/systemd/system/finance-daily.service
```

### 完整配置示例

```ini
[Unit]
Description=Finance Daily Backend
After=network.target

[Service]
Type=simple
WorkingDirectory=/var/www/finance-daily/backend
Environment="PATH=/usr/bin:/usr/local/bin:/var/www/finance-daily/backend/venv/bin"
ExecStart=/var/www/finance-daily/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

### ⚠️ 关键注意事项

| 配置项 | 说明 | 重要性 |
|--------|------|--------|
| `PATH` | **必须包含 `/usr/bin`**，否则 ffmpeg/ffprobe 等系统命令找不到 | ⭐⭐⭐⭐⭐ |
| `WorkingDirectory` | 后端代码目录 | ⭐⭐⭐⭐ |
| `Restart=always` | 服务崩溃后自动重启 | ⭐⭐⭐ |

### 为什么本地没问题，服务器会出错？

| 环境 | PATH 来源 | 结果 |
|------|----------|------|
| **本地** | 继承系统完整 PATH（自动包含各种命令目录） | ✅ 正常 |
| **服务器 systemd** | 需要手动在配置文件中指定 | ❌ 如果漏掉会找不到命令 |

### 换服务器时的注意事项

这个配置文件**不在 Git 仓库里**，换新服务器需要重新创建：

```bash
# 1. 创建配置文件
sudo nano /etc/systemd/system/finance-daily.service

# 2. 粘贴上面的完整配置

# 3. 重载并启动
sudo systemctl daemon-reload
sudo systemctl enable finance-daily
sudo systemctl start finance-daily
```
